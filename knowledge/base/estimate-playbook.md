# Estimate Playbook — 代理店視点の見積り・工数計算

代理店・受託の提案書、社内施策の予算承認、コンペ向けラフ概算など、**「いくらかかるか」を根拠付きで示す**ための実務プレイブック。

RMC 上の位置づけは Insight（経営承認前の試算）→ Activate / Learn（提案書・見積書として出力）。見積りと実際の工数・媒体費・ROAS の差分は `/learn` 経由で `memory/results/` に戻し、次回見積りの単価・係数較正に使う。

## 見積りの基本構造

見積りは **3 層**に分けると顧客が理解しやすい：

1. **制作費（イニシャル）** — 一度きりの作業（戦略設計 / クリエイティブ / LP 構築 / 広告入稿）
2. **運用費（ランニング・月額）** — 継続作業（運用最適化 / レポーティング / 定例 MTG / クリエイティブ更新）
3. **媒体費・外部費用** — 広告出稿費 / ツール利用料 / 撮影・外注費

**作業費と媒体費を分離する**のが鉄則。混ぜると顧客が比較・判断できない。

## 職種別の標準単価レンジ

参考値。地域・規模・経験で変動するため、自社の実績で較正していく。

| コピーライター | ¥40,000〜80,000 | 広告コピー、記事、LP 文面 |

### 工数積算のルール

- **1 人日 = 8 時間稼働**
- **バッファ係数**: 工数 × 1.2〜1.5（リスクに応じて）
- **クライアント確認待ちは工数に含めない**（スケジュールには反映）
- **修正回数の前提を明示**（例: デザイン修正 2 回まで含む）

## 松竹梅（3 段プラン）パターン

提案書・コンペでは詳細見積りを出す前に**スコープの異なる 3 案**を提示する。中央値（竹）を選ばせる設計：

```markdown
## 概算: [施策名]

| プラン | 内容 | 概算費用（税抜） | 期待効果 |
|--------|------|-----------------|---------|
| 松（フルスコープ） | ... | ¥X,XXX,XXX 〜 | ... |
| 竹（標準・推奨） | ... | ¥X,XXX,XXX 〜 | ... |
| 梅（ミニマム） | ... | ¥XXX,XXX 〜 | ... |

※ 正式な見積りは詳細ヒアリング後に提出
```

設計原則：
- **竹を中心に**売れる設計（最頻 80% は竹）
- **松は 1.5〜2 倍**、梅は **0.5〜0.7 倍** が目安（差が開きすぎると分断感）
- **梅は失注時の最低ライン** — 「その予算ならこれだけはできる」を示す保険

## ROI シミュレーション

予算承認・ピッチ用。**悲観 / 標準 / 楽観**の 3 シナリオで損益分岐点を示す：

```markdown
## ROI シミュレーション

### 前提条件
| 指標 | 値 | 根拠 |
|------|-----|------|
| 総投資額 | ¥X,XXX,XXX | 見積り合計 |
| 媒体費 | ¥X,XXX,XXX | 上記のうち |
| 想定 CPC | ¥XXX | 業界平均 / 過去実績 |
| 想定 CTR | X.X% | 業界平均 / 過去実績 |
| 想定 CVR | X.X% | 業界平均 / 過去実績 |
| 平均客単価 | ¥XX,XXX | 実績ベース |

### 試算結果
| シナリオ | CV 数 | 売上 | ROAS | CPA |
|----------|------|------|------|-----|
| 悲観 | XXX | ¥X,XXX,XXX | X.X | ¥X,XXX |
| 標準 | XXX | ¥X,XXX,XXX | X.X | ¥X,XXX |
| 楽観 | XXX | ¥X,XXX,XXX | X.X | ¥X,XXX |

### 損益分岐点
- 必要 CV 数: XXX 件
- 必要 CVR: X.X%
- 投資回収期間: X ヶ月（標準シナリオ）
```

自社実績データが無い場合は業界平均で埋めた上で、**「自社データ不足」を必ず明示**する。架空値で予算承認を取らない。

## Principles

- **根拠のある数字** — 「だいたいこのくらい」ではなく、工数 × 単価の積み上げで算出
- **前提条件を明示** — 修正回数 / 対応範囲 / 納品物 / 確認待ち期間を必ず書く
- **松竹梅を示す** — 1 案だけでなくスコープの異なる複数案を提示して選択肢を作る
- **バッファを忘れない** — 工数 × 1.2〜1.5、想定外の作業に備える
- **媒体費と作業費を分離** — 顧客が費目を理解できるように
- **マージンは透明に** — 代理店手数料の考え方を隠さない（不信を招く）
- **自社実績が無い場合は明示** — 業界平均で埋めた前提を黙って使わない

## RMC との接続

- **Set**: `business-overview.md` の収益モデル、`performance-data.md` の CPC / CTR / CVR ベンチマーク、自社の単価実績を蓄積
- **Ask**: 「LP 制作 + Google 広告 3 ヶ月運用の松竹梅見積り」「この施策の ROI 損益分岐点を試算」
- **Release**: 提案書 / 見積書を実際にクライアントに提出 / 社内承認に出す
- **Feedback**: 見積り vs 実際の差分（工数・媒体費・ROAS）を `memory/results/` に記録 → 次回見積りの単価・係数較正
